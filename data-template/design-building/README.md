# 建筑数据模板说明

`模板-建筑` 用于 `design-building` 中建筑文档的最大兼容结构化格式。

## 格式规则

- 第一行写建筑名，作为文档标题与目录名的一致主键。
- 字段统一使用中文全角冒号 `：`。
- 字段和区块顺序固定，保持：
  - 建筑名
  - `攻击类型`
  - `攻击距离`
  - `生命`
  - `攻击`
  - `护甲`
  - `魔抗`
  - `回血`
  - `攻击间隔`
  - `击杀奖励`
  - `被动`
  - `特殊效果`
- `被动` 与 `特殊效果` 放在独立区块，区块内部允许多行说明文本；避免无意义换行破坏字段解析。

## 兼容性规则（用于标准化脚本）

- `防御塔i` 到 `防御塔iv` 与 `遗迹`、`兵营` 的字段取并集后得到上述顺序。
- `防御塔ii` 的文件名为 `防御塔ii`，如标题出现旧的 `防御塔ii三` 将按文件名回写为 `防御塔ii`。
- 旧文档中如果有“真视/视野”一类独立效果文本，会被归入 `特殊效果`。

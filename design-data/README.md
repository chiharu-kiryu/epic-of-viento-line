# Design Data 规范

`design-data/` 是本仓库的主资料区，目前共包含约 `507` 份设计文档，按内容类型拆分如下：

| 目录 | 数量 | 说明 |
| --- | ---: | --- |
| `backstory/` | 122 | 英雄背景故事，按力量/敏捷/智力分类。 |
| `design-heros/` | 123 | 英雄数值与技能设计文档。 |
| `design-item/` | 154 | 装备与消耗品设计文档。 |
| `design-building/` | 6 | 建筑设计文档。 |
| `design-rules/` | 28 | 战斗与成长规则说明。 |
| `design-scenes/` | 19 | 场景与地图资料。 |
| `design-skills/` | 18 | 通用召唤师技能设计。 |
| `design-units/` | 32 | 中立单位与兵线单位设计。 |
| `design-template/` | 2 | 模板文档。 |

## 通用写作规则

- 第一行写文档主标题，一般直接使用实体名称。
- 段落之间保留空行，避免把不同语义块挤在一起。
- 数值字段统一使用 `字段名：数值` 形式。
- 同一类文档应尽量保持固定字段顺序。
- 技能、被动、效果说明优先按“名称在上，描述在下”的方式书写。
- 若使用简称、别名或世界观称谓，建议在标题后尽快给出正式名称。

## 各类文档建议格式

### `backstory/`

适用于文学化背景故事。

- 标题使用角色名。
- 正文保持自然分段，每段聚焦一个叙事片段。
- 若需要引用台词，单独成段。

### `design-heros/`

建议字段顺序：

1. 英雄名
2. 别名或称号
3. 攻击类型
4. 基础属性与成长
5. 天生技能
6. 技能 1 到技能 4
7. 阳印 / 阴印 / 铸魔 / 铸神

建议字段名：

- `攻击距离：`
- `基础攻击间隔：`
- `基础移动速度：`
- `力量：`
- `敏捷：`
- `智力：`
- `天生技能：`
- `技能1：` 到 `技能4：`

属性成长推荐写成 `20 + 2.1` 的形式，避免与技能描述混排。

### `design-item/`

建议字段顺序：

1. 装备名
2. 属性收益
3. 价格
4. 主动或被动效果
5. 风味描述

建议字段名：

- `价格：`
- `主动：`
- `被动：`
- `唯一被动：`

### `design-building/`

建议字段顺序：

1. 建筑名
2. 生命
3. 护甲
4. 魔抗
5. 回复
6. 击杀奖励

建议统一使用：

- `生命：`
- `护甲：`
- `魔抗：`
- `回血：`
- `击杀奖励：`

### `design-rules/`

规则文档内容差异较大，但建议至少保证：

- 首行是规则主题。
- 公式、阈值、例外情况分段说明。
- 若存在公式，优先单独一行展示。
- 若规则存在前置名词定义，先定义再说明计算方式。

### `design-scenes/`

建议包含以下信息：

1. 场景名
2. 场景定位
3. 地貌或区域特征
4. 玩法关联
5. 与其他地图连接关系

`地图连接表.json` 作为结构化数据文件保留 JSON 格式，不混入自然语言注释。

### `design-skills/`

建议字段顺序：

1. 技能名
2. 类型
3. 效果描述
4. 冷却
5. 消耗
6. 驱散、命中或特殊判定

建议字段名：

- `类型：`
- `冷却：`
- `魔力消耗：`

### `design-units/`

建议字段顺序：

1. 单位名
2. 基础面板
3. 击杀奖励
4. 被动
5. 主动

建议字段名：

- `生命：`
- `攻击：`
- `护甲：`
- `魔抗：`
- `攻击间隔：`
- `移动速度：`
- `击杀奖励：`
- `被动：`
- `主动：`

## 命名与目录约定

- 英雄、单位、装备、技能、素材名称尽量保持一一对应。
- 不在文档文件名中混入版本号、日期或“最终版”等临时标记。
- 临时整理内容放在 `temps/`，稳定后再迁回正式目录。
